DOM Structure

To access the DOM tree, you would need to start from the top, which is in this case is the document object. This document object has properties or method (function that is inside an object.) inside that allows users of javacript to access all the elements inside of it.

Notes

Selecting the elements inside the DOM

To access these elements or nodes of the DOM tree, we first need to convert them into an object inside the javascript, for us to be able to manipulate them. There are 5 main ways of seleccting an element which are: 

              
              <body>
                <h1 id="myHeader">My heading</h1>
                  <ul class="myList">              
                    <li class="myListItem"> Item 1 </li>
                    <li class="myListItem"> Item 2 </li>
                    <li class="myListItem"> Item 3 </li>
                  </ul>
              </body>

GetElementByID()

- This method uses the unique Id of an element in order to select. The way we use it is by declaring a variable and assigning the element into it so that we can manipulate it later. We first start with the document object then we traverse to the specific widget we want to get. 

          
            const elementWithAnId = document.getElementById('myHeader');

GetElementByClassName()

- This method uses the class of an element in order to select. The way we use it is similar to how we use the .getElementById(). The only difference is that since this class is not unique compared to an Id, when we use this, this can return an array of elements containing that class. It is sorted by the order they appear in the HTML file. 

          
            const elementWithAClass = document.getElementByClassName('myList');
            const listItems = document.getElementByClassName('myListItem')

            //if you console log both:
            console.log(elementWithAClass); //Will return the HTML
            console.log(listItems); //Will return an array of html element.

GetElementByTagName()

- This method uses the Tag itself of an element in order to select. The way we use it is similar to how we use the .getElementClassName() and can also return an array-like list of tags selected by it sorted in the order they appear on the HTML 

          
            const elementWithH1Tag = document.getElementByTagName('h1');
            const elementWithULTag = document.getElementByTagName('ul');
            const elementWithLITag = document.getElementByTagName('li');

querySelector()

This method lets you grab an element using the same way you target things in CSS.

  • # → for id
  • . → for class
  • tag name → for elements like li, div, etc.

⚠️ Important: It only returns one element — the first match it finds.

👉 Example: If you select li, it will only get the first list item (Item 1), not all of them. 

          
            const firstLI = document.querySelector('li');
            const firstMyListItem = document.querySelector('.myListItem');
            const myh1 = document.querySelector('#myheader');

querySelectorAll()

- This method is the same as .querySelector but selects all of the element it matches. This returns an array-like list. 

          
            const allLI = document.querySelectorAll('li');
NOTES:
  • I would suggest querySelectorAll or querySelector as this can target any element. The main flaw is that other browsers might not be able to support it. But as of writing, it is now 2026, so I guess most modern browsers can use it.
  • Use the other 3 as older browsers can support it.

Styling the element

One way to manipulate the DOM is to style its element. We need to get the element itself using the any of the 5 main methods of selecting the element and assigning it to a variable. We will be using the same sample html to be consistent. 

              
              <body>
                <h1 id="myHeader">My heading</h1>
                  <ul class="myList">              
                    <li class="myListItem"> Item 1 </li>
                    <li class="myListItem"> Item 2 </li>
                    <li class="myListItem"> Item 3 </li>
                  </ul>
              </body>

After selecting the element, we will apply the style property (method) and use any css property. You can find the list here. Take note that if it consist of two words, we will use camel case instead kebab case which is what we usually use in CSS. 

                  
                    const myHeader = document.querySelector('#myHeader');

                    myHeader.style.color = 'red'; //The text color will turn to red.
                    myHeader.style.fontSize = '2rem' //The text size will change its size into 32px. Default size of 1rem is 16px.

NOTES: Please take note of what method you are using to style an element. In this case, we are using what is called an inline-styling, which can usually style one element only. Let's say we target all the 'li' elements. 

                  
                    const firstLI = document.querySelector('li');

                    firstLI.style.color = 'red'; //Only the first LI will have a styling of red.

                    //Even if you use the .querySelectorAll,

                    const allLI = document.querySelectorAll('li');

                    allLI.style.color = 'red'; //This will only throw a TypeError since we are styling an array, not a string.

To style all the li elements, we need to iterate/loop all of the li that is returned in an array form when we use querySelectorAll.

                  
                    const allLI = document.querySelectorALL('li');

                    allLI.forEach(element => {
                      element.style.color = 'red';
                    }); //This will make sure that all the li element text color will change into red.

Creating an element

Creating an element is also one of the way we can manipulate the DOM. One use of creating an element is when we are creating a todo app. Here are the ways to create an element. You can also read more here. For now let us declare all the elements we need.

  • .append(): We use this method to tell javascript that we want to attached the created li element to the ul element, which will make the ul element its parent.

Modifying the text

We can change the text, image or video of an element. There are three ways to do this:

  • .innerText: returns the text content of the element and all descendants, with spacing and CSS hidden text, but without tags.
  • .textContent: returns just the text content of the element and all its children, without CSS hidden text spacing and tags, except <script> and <style> elements.
  • .innerHTML: returns the text content of the element, including all spacing and inner HTML tags.
              
              <body>
                <h1 id="myHeader">My heading</h1>
                  <ul class="myList">              
                    <li class="myListItem"> Item <p>1</p> </li>
                    <li class="myListItem"> Item 2 </li>
                    <li class="myListItem"> Item 3 </li>
                  </ul>
                  <script>
                    const firstMyListItem = document.querySelector('myListItem');

                    console.log(firstMyListItem.innerText);//returns: Item 1
                    console.log(firstMyListItem.textContent);//returns: Item 1, the "1" wrapping to the next line
                    console.log(firstMyListItem).innerHTML;//returns: Item <p>1</p>

                    firstMyListItem.innerText = 'Item 4'; // Changes the text from Item 1 to Item 4.
                  </script>
              </body>

Explanation

  • We get the first list item in the list from the HTML document and turn it into a JavaScript object by assigning it into the firstMyListItem variable
  • We then attached the .innerText to the firstMyListItem variable the assigned the value Item 4 to it so the text content changes.

NOTES: It is usually not advisable to use .innerHTML because of security reasons. That is why most users prefer the other two.

Modifying the Attributes and Classes

Once we created an element, to apply the styling similar to the other list items, we need to modify its attributes and classes. We can do that using the .setAttribute method. To remove an attribute, we use the .removeAttribute method.

The .classList is another method to modifying an attribute. We can utilize the .add and .remove method.

To find out if an element has a certain attribute, we can use the .contains() method. This will return a boolean expression.

              
              <body>
                <h1 id="myHeader">My heading</h1>
                  <ul class="myList">              
                    <li class="myListItem"> Item <p>1</p> </li>
                    <li class="myListItem"> Item 2 </li>
                    <li class="myListItem"> Item 3 </li>
                  </ul>
                  <script>
                    const fourthMyListItem = document.createElement('li');
                    const fifthMyListItem = document.createElement('li');

                    fourthMyListItem.innerText = 'Item 4'; // Assigns the text content to Item 4.
                    fourthMyListItem.setAttribute('class', 'myListItem');

                    fifthMyListItem.innerText = 'Item 5'; // Assigns the text content to Item 5.
                    fifthMyListItem.classList.add('myListItem');
                    fifthMyListItem.classList.contains('myListItem');//returns'true' to the console.
                  </script>
              </body>

Explanation

  1. We declare an variable and assigned the li element to it.
  2. We assigned the text Item 4 to the fourth li element
  3. To ensure that the styling applies to the fourth li, we add the class attribute myListItem.

NOTES: We can also use the .getAttribute method to get the attribute of an element.

Lesson 4: Traversing the DOM

Let's take a look at the DOM Tree below. The Document element is the root element. To traverse the DOM, we can use the relationship between elements to navigate easily. The relationship can be parent-child relationship or sibling relationship. There are other relationship which you can find in google(Yeah, search for it yourself).

Elements that are above another element is a parent of that element. Alternatively, an element that is directly below an element is its child.

  • Document object is the parent of the <HTML> element.
  • <HTML> element is the parent of the <head> and <body> tag.
  • <p> and <h1> tag are all the child of the <body> element, which can also be called its parent

Elements that are on the same level of an element is a sibling of that element.

  • <head> and <body> element are siblings.
  • <p> and <h1> element are also siblings.

Using our example, let's establish the parent-child relationship and sibling relationship. I added a title to the list to expand our example.

              
              <body>
                <h1 id="myHeader">My heading</h1>
                  <p> Item List </p>
                  <ul class="myList">              
                    <li class="myListItem"> Item 1 </li>
                    <li class="myListItem"> Item 2 </li>
                    <li class="myListItem"> Item 3 </li>
                  </ul>
              </body>

Relationship

  1. Parent-Child elements
    • <h1> and <p>
    • <h1> and <ul>
    • <ul> and all <li>
  2. Sibling elememts
    • <p> and <ul>
    • all <li>

Parent Node Traversal

We will start at the <ul> tag, we will then go up to its parent using these two methods: .parentNode and .parentElement. We will use this to go up the document. 

            
            <body>
              <h1 id="myHeader">My heading</h1>
                <p> Item List </p>
                <ul class="myList">              
                  <li class="myListItem"> Item 1 </li>
                  <li class="myListItem"> Item 2 </li>
                  <li class="myListItem"> Item 3 </li>
                </ul>
                <script>
                  const myUl = document.querySelector('.myList');

                  console.log(myUl.parentNode); 
                  console.log(myUl.parentElement);
                  //Both will log the h1 element.

                  console.log(myUl.parentNode.parentNode); 
                  console.log(myUl.parentElement.parentElement);
                  //Both will log the body element.
                </script>
            </body>

NOTES:
Whats the difference between the two?

  • parentNode: this is a readonly property. Meaning you cannot change its content.This will return any node, regardless of type.
  • parentElement: editable,which can take the innerText property etc., to alter the content, color, size, etc. This will only return an element. Lets say you are at the <html> element, calling a .parentElement will return a null on the console. The .parentNode will return the document node itself.

Additionally, you can stack these two methods to get to the grandparent element.

Child Node Traversal

To go down the document, we will use the child node traversal. We will still use the <ul> tag as a starting point. Similar to parent node traversal, we also have .childNode and .children methods. A video explanation could be found here. Others: .firstChild, .lastChild, .firstElementChild, .lastElementChild 

            
            <body>
              <h1 id="myHeader">My heading</h1>
                <p> Item List </p>
                <ul class="myList">              
                  <li class="myListItem"> Item 1 </li>
                  <li class="myListItem"> Item 2 </li>
                  <li class="myListItem"> Item 3 </li>
                </ul>
                <script>
                  const myUl = document.querySelector('.myList');

                  console.log(myUl.childNode); 
                  //Will log an array of nodes(element nodes, comment nodes, text nodes).
                  myUl.childNode[1].style.color = 'blue';
                  //this will make the target element's text turn into blue.

                  console.log(myUl.firstElementChild); 
                  console.log(myUl.lastElementChild).
                  //Will log the first element and last element respectively.
                </script>
            </body>

Remember since the .childNode method returns an array, if we want to style an element, we need to access that element the same way we access a normal array.

NOTES:
HTML Nodes vs Elements

      childNodes vs children

    • childNodes: returns child nodes (element nodes, text nodes, and comment nodes).
    • children: returns child elements (not text and comment nodes).

      firstChild vs firstElementChild

    • firstChild: returns the first child nodes (an element node, a text node or a comment node). Whitespace between elements are also text nodes.
    • firstElementChild: returns the first child elements (not text and comment nodes).

      lastChild vs lastElementChild

    • lastChild: returns the last child nodes (an element node, a text node or a comment node). Whitespace between elements are also text nodes.
    • lastElementChild: returns the last child elements (not text and comment nodes).

Sibling Node Traversal

An element will become a sibling of another element if they are on the same level under the same parent. Additionally, it is not limited to to elements, comment nodes and other nodes can become a sibling of an element as long as the condition is met.

Methods to traverse the DOM between siblings:

  • .previousSibling: returns the prevous node on the same tree level in the form of a node object that is read-only.
  • .nextSibling: returns the next node on the same tree level. It returns a node object and is read-only.
  • .previousElementSibling: returns the previous element in the same tree level and is read-only.
  • .nextElementSibling: returns the next element in the same tree level.It is a read-only property.

NOTES:

It is prefered to use the .previousElementSibling and .nextElementSibling because it only returns the element unlike the previousSibling and nextSibling which return the other nodes(text, comment, element) as a node object.

Event object

The Event Object (e) is automatically created whenever an event occurs (e.g., click, hover, keypress). It contains useful information about the event and allows us to interact with the DOM based on what happened.

Common Properties:

  • e.target – the actual element that triggered the event
  • e.currentTarget – the element where the event listener is attached
  • e.type – the type of event (click, keydown, etc.)
  • e.timeStamp – the time (in ms) when the event occurred

Mouse Event Properties:

  • e.clientX – X position of the mouse (viewport)
  • e.clientY – Y position of the mouse (viewport)
  • e.pageX – X position relative to the whole page
  • e.pageY – Y position relative to the whole page

Keyboard Event Properties:

  • e.key – the key pressed (e.g., "a", "Enter")
  • e.code – physical key on the keyboard (e.g., "KeyA")
  • e.ctrlKey – true if Ctrl is pressed
  • e.shiftKey – true if Shift is pressed
  • e.altKey – true if Alt is pressed

Useful Methods:

  • e.preventDefault() – stops default behavior (e.g., form submit, link navigation)
  • e.stopPropagation() – stops the event from bubbling up

The event object is passed as a parameter to event handler functions.

Event Propagation

Event Propagation refers to how an event travels through the document object model or DOM tree.

A similar comparison is electricity traveling to a wire until it reaches it destination. A DOM tree consist of nodes, so the event travels every node until it reach the target element which is then it is forcibly stopped. It has 3 phases:

  1. Event Capturing
  2. Target
  3. Event Bubling

This phase starts from the Event Capturing, which starts from root(document) and travels all the way to the target. When it reaches the target, it starts the Event bubbling phase, which begins from the target, all the way back to the root.

Event Delegation in Action

The code that actually makes the page scroll on the specific content when you click the topic in the sidebar was actually made using event delegation and the method .scrollIntoView

            
                <script>
                  document.querySelector("#lesson-list").addEventListener("click", (e) => {
                  const idName = e.target.getAttribute("id");
                  const mainDivs = document.querySelector("main").children;

                  for (let div of mainDivs) {
                    if (div.classList.contains(idName)) {
                      div.scrollIntoView({
                        behavior: "smooth",
                        block: "start",
                      });
                    }
                  }
                });
                </script>

The algorithm is as follows:

  1. Select the parent element that contains all the topics in the sidebar. In this case, it is #lesson-list.
  2. Attach a click event listener to this parent element.
  3. Pass the event object e into the callback function to access information about the triggered event.
  4. Retrieve the id of the element that was clicked using e.target, and store it in the variable idName.
  5. Select all child elements inside the main element and store them in mainDivs. These represent the content sections.

Intersection Observer API

What if there is a way to track when an element enters the screen? That way, we can attach animations the moment it does enter the screen and leave outside of the screen. Introducting Intersection Observer API

Notice that when you scroll, the cards slides from the left and stays at that position and slides out when it exits the screen. This is made using that API.

            
                <script>
                  const cards = document.querySelectorAll(".card");

                  const cardObserver = new IntersectionObserver(
                    function (entries) {
                      entries.forEach((entry) => {
                        entry.target.classList.toggle("show", entry.isIntersecting);
                      });
                    },
                    { threshold: 0.07 },
                  );

                  cards.forEach((card) => {
                    cardObserver.observe(card);
                  });
                </script>

  1. Select the parent element that contains all the topics in the sidebar. In this case, it is #lesson-list.
  2. Attach a click event listener to this parent element.
  3. Pass the event object e into the callback function to access information about the triggered event.
  4. Retrieve the id of the element that was clicked using e.target, and store it in the variable idName.
  5. Select all child elements inside the main element and store them in mainDivs. These represent the content sections.
  6. Loop through mainDivs using a for...of loop. For each element, check if its class matches idName.
